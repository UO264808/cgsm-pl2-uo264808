Memoria

Descripción de la práctica

En esta sesión se han desarrollado una serie de ejercicios con el fin de ejemplificar la utilización de WebRTC para realizar llamadas de audio y video.
Esta tecnología permite integrar RTC (Real Time Communications) en el navegador sin requerir ningún software adicional, permitiendo el desarrollo de aplicaciones multiplataforma de una forma muy simple.

Ejercicio 1

En este primer ejercicio se realiza una página web que permite capturar los frames de la cámara web del equipo. En el script de Javascript asociado a dicha página se incluye código que permite activar la cámara web y redirige el flujo a un elemento video del documento html.

Ejercicio 2

En este ejercicio, se toma como punto de partida el ejercicio anterior, al que se le añadirá la posibilidad de guardar a disco algún frame capturado por la cámara, como si de una cámar de foto se tratase.

Ejercicio 3

El objetivo de este ejercicio es lograr la comunicación en tiempo real dentro de una misma página web, ilustrando el funcionamiento de PeerConnection. Resulta de suma importancia la implementación de tres funciones:

  1. Start: Función que permite arrancar el servicio de captura de video local.
  2. Call: Función encargada de enviar los flujos de datos a remote.
  3. Hangup: Función cuya misión es finalizar el servicio de llamada.

Ejercicio 4

Este último ejercicio se trata de una modificación del ejercicio 2 de la práctica 2 en el que aparecía en la escena un rectángulo girando sobre si mismo mostrando un video. En estecaso la variación será que la textura en vez de ser un video será que la textura se obtenga a partir de lo capturado con la cámara web de equipo.

Dificultades encontradas

La realización de la práctica trnasucurrió sin grandes dificultades. Reseñando tan sólo las funciones a implementar del ejercicio 3 debido a una mayor complejidad de conceptos respecto a los demás ejercicios.
